A Different View on Software Development
CUBE
From his years of experience in IT, Johan Meijdam has developed his own vision on system development. The CUBE development framework is the implementation of this vision.
It is common in company-wide software development to start making the Entity Releationship Diagram (ERD), on which the business processes are defined. In contrast to the ERD, CUBE uses a hierarchical business object model.
An Agile/Scrum approach requires a flexible solution for system development. CUBE provides a repeatable process whereby the hierarchical business object model is automatically translated into code in different steps according to self-developed templates.
For more detail check the CUBE page.
CubeGen
A part of the CUBE development framework is the open code generator. CubeGen has been developed especially for this purpose. The tool is so generic that it can also be used for other applications.
For more detail check the CubeGen page.
Johan Meijdam
As a senior information architect, Johan Meijdam has a broad knowledge and long experience in the field of technical and administrative information processing.
Since 1981 he has been involved in consultancy, architecture, project management, analysis, design and realization.
He specializes in Model Based Application Development. An experience built up in a CA Gen (formely COOL:gen, IEF) environment. He developed his own vision on software development, which he now makes concrete with the development of the CUBE development framework.
